## 1. React Native Project Setup

### Version & Initialization

**React Native 0.80.x** with TypeScript (default in 0.71+).

```bash
# Initialize with RN CLI (not Expo) because we need native modules:
# - Geolocation (native iOS/Android location APIs)
# - Maps (native map views)
# - Camera access

npx react-native@0.80.2 init JourneyTogether --version 0.80.2
cd JourneyTogether
```

**Why not Expo?** Expo would work for most of this, but:
- `@react-native-community/geolocation` has better background location support
- `react-native-maps` requires native setup that Expo prebuild handles but adds complexity
- For self-contained APK builds, bare RN gives more control

**Alternative:** Use Expo with `npx expo prebuild` after init — both approaches converge. The plan below is compatible with either; just swap native packages for their Expo equivalents (`expo-location`, `expo-image-picker`, `react-native-maps` → `react-native-maps` via Expo).

### Environment / Config

**⚠️ CRITICAL: Backend URL must be configured at build time.** The APK is a self-contained bundle that users install on their phones. When users are on cellular (not on the same LAN as the Unraid server), a hardcoded LAN IP like `192.168.1.200` will fail. The APK must connect to **the user's own Unraid server**.

**Build-time configuration approach:**
- Set `API_BASE_URL` and `WS_BASE_URL` via environment variables at build time
- For debug builds: use `.env` file or gradle properties
- For release/production APK: inject the user's server URL during the CI/CD build
- The URL should be the **public-facing domain or IP** of the user's Unraid server (e.g., `https://journey.example.com` or `https://unraid.example.com:3000`)

**Android gradle configuration (`android/app/build.gradle`):**
```groovy
// Inject API_BASE_URL and WS_BASE_URL from gradle properties or env
def apiBaseUrl = project.hasProperty('API_BASE_URL') ? project['API_BASE_URL'] : 'http://192.168.1.200:3000'
def wsBaseUrl = project.hasProperty('WS_BASE_URL') ? project['WS_BASE_URL'] : 'ws://192.168.1.200:3000'

// Expose to JS via BuildConfig
buildConfigField "String", "API_BASE_URL", "\"${apiBaseUrl}\""
buildConfigField "String", "WS_BASE_URL", "\"${wsBaseUrl}\""
```

**CI/CD pipeline (example GitHub Actions):**
```yaml
# In your CI/CD pipeline that builds the APK
- name: Build Android APK
  env:
    API_BASE_URL: ${{ vars.API_BASE_URL }}  # User's server URL
    WS_BASE_URL: ${{ vars.WS_BASE_URL }}
  run: |
    cd android
    ./gradlew assembleRelease \
      -PAPI_BASE_URL="$API_BASE_URL" \
      -PWS_BASE_URL="$WS_BASE_URL"
```

**`src/config/env.ts`:**
```ts
// API base URL — read from BuildConfig (set at build time by gradle)
// NEVER hardcode a LAN IP in production builds

const getEnvVar = (key: string, fallback: string): string => {
  // In Expo: use expo-constants
  // In bare RN: use react-native-config or BuildConfig
  try {
    // BuildConfig is auto-generated by gradle from android/app/build.gradle
    return (global as any).BuildConfig?.[key] ?? fallback;
  } catch {
    return fallback;
  }
};

export const API_BASE_URL = getEnvVar('API_BASE_URL',
  __DEV__ ? 'http://192.168.1.200:3000' : 'http://192.168.1.200:3000'
);

export const WS_BASE_URL = getEnvVar('WS_BASE_URL',
  __DEV__ ? 'ws://192.168.1.200:3000' : 'ws://192.168.1.200:3000'
);

export const UPLOADS_BASE_URL = `${API_BASE_URL}/uploads`;
```

**CI/CD users:** Set `API_BASE_URL` and `WS_BASE_URL` to your public-facing Unraid server URL in your CI/CD environment variables or gradle properties before building the APK.

## 2. State Management (Zustand)

### Store Architecture

Four Zustand stores, each focused on a single domain:

```
stores/
├── authStore.ts       # User session, JWT, profile
├── journeyStore.ts    # Journey list, current journey details, search/filter
├── sessionStore.ts    # Active journey session (solo + group)
└── uiStore.ts         # Global UI state (modals, loading, toasts)
```

### authStore.ts

```ts
interface User {
  id: string;
  email: string;
  display_name: string;
  avatar_url: string | null;
  role: 'user' | 'creator' | 'admin';
  created_at: string;
}

interface AuthState {
  user: User | null;
  token: string | null;
  isLoading: boolean;       // Initial auth check in progress
  isAuthenticated: boolean;

  // Actions
  login: (email: string, password: string) => Promise<void>;
  register: (email: string, password: string, displayName: string) => Promise<void>;
  logout: () => Promise<void>;
  loadToken: () => Promise<void>;   // Boot: read from AsyncStorage, validate
  updateProfile: (updates: Partial<User>) => Promise<void>;
  uploadAvatar: (uri: string) => Promise<void>;  // MAJOR: avatar upload
  setUser: (user: User) => void;
}
```

**Storage:** `token` and `user` persisted to `@react-native-async-storage/async-storage` under keys `jt_token` and `jt_user`.

**Interactions:** `sessionStore` consumes `authStore` to get the JWT for API calls. Navigation guards check `isAuthenticated`.

### sessionStore.ts

```ts
interface SessionMember {
  user_id: string;
  display_name: string;
  avatar_url: string | null;
  completed_stop_ids: string[];
  current_stop_index: number;
}

interface ActiveSession {
  id: string;
  journey_id: string;
  journey: Journey;           // Denormalized for offline access
  owner_id: string;
  invite_code: string | null; // null = solo session
  status: 'waiting' | 'active' | 'paused' | 'completed';
  current_stop_index: number;
  members: SessionMember[];
  myProgress: {
    completed_stop_ids: string[];
    photos: Photo[];
    reactions: Record<string, string>; // stopId -> emoji (MAJOR: per-member reactions via WS)
  };
  created_at: string;
}

interface Photo {
  id: string;
  url: string;
  stop_id: string;
  uploaded_by: string;
  created_at: string;
}

interface SessionState {
  activeSession: ActiveSession | null;
  wsConnection: WebSocket | null;
  isConnecting: boolean;

  // Actions
  startSoloSession: (journeyId: string) => Promise<void>;
  startGroupSession: (journeyId: string) => Promise<void>;  // Returns invite_code
  joinSession: (inviteCode: string) => Promise<void>;
  leaveSession: () => Promise<void>;
  completeStop: (stopId: string, location: { lat: number; lng: number }) => Promise<void>;  // CRITICAL: location now sent to server for Haversine validation
  uploadPhoto: (stopId: string, uri: string) => Promise<void>;
  uploadVoiceNote: (sessionId: string, stopId: string, uri: string) => Promise<void>;  // MAJOR: voice note upload
  pauseSession: () => Promise<void>;   // MAJOR: pause a journey
  resumeSession: () => Promise<void>;  // MAJOR: resume a paused journey
  addReaction: (stopId: string, emoji: string) => Promise<void>;  // MAJOR: add reaction to stop
  syncSession: () => Promise<void>;   // Re-fetch session from server

  // WebSocket
  connectWS: (sessionId: string) => void;
  disconnectWS: () => void;
  handleWSMessage: (message: WSMessage) => void;
}

type WSMessage =
  | { event: 'session_state'; data: { session: any; journey: Journey; members: SessionMember[]; photos: Photo[] } }
  | { event: 'member_joined'; data: SessionMember }
  | { event: 'member_left'; data: { user_id: string } }
  | { event: 'stop_completed'; data: { user_id: string; stop_id: string; stop_index: number; all_at_stop: number; total_members: number; journey_completed: boolean; new_current_stop_index: number } }
  | { event: 'journey_completed'; data: { session_id: string; completionId: string } }
  | { event: 'photo_added'; data: { photo: Photo } }
  | { event: 'reaction_added'; data: { user_id: string; stop_id: string; emoji: string } }  // MAJOR: shown on MemberAvatar badge
  | { event: 'session_paused'; data: { session_id: string } }
  | { event: 'session_resumed'; data: { session_id: string } }
  | { event: 'error'; data: { code: string; message: string } };
```

**Interactions:** `sessionStore` is the most complex. It:
- Manages WebSocket lifecycle (connect on join, disconnect on leave)
- Updates `activeSession` from both REST API responses and WS events
- Exposes computed values like `allMembersCheckedIn` (for group sync), `currentStop`, `nextStop`
- `authStore.token` is required for all API calls within sessionStore actions

### HomeStack

#### HomeScreen
- **Purpose:** Journey discovery feed
- **Layout sections:**
  1. Search bar (`SearchBar` component) — triggers `journeyStore.setSearchQuery` + debounced fetch
  2. Tag filter chips (`TagChip` horizontal scroll) — `journeyStore.setTagFilter`
  3. **"Near me" toggle** (MAJOR: Spontaneous Mode) — when enabled, uses `getCurrentPosition()` to get user location, then calls `GET /journeys?lat=&lng=&radius_km=5`. Shows nearby journeys instead of all journeys. Each nearby journey card displays distance from user (e.g. "1.2 km away"). Uses `journeyStore.setNearbyEnabled()` and `journeyStore.fetchNearbyJourneys(location)`.
  4. "Featured" horizontal carousel (only shown if `featuredJourneys.length > 0` and nearby mode is OFF)
  5. **"Nearby Journeys" section** (only shown when nearby toggle is ON and `nearbyJourneys.length > 0`) — list or map of public journeys within ~5km, each showing distance badge
  6. "All Journeys" vertical list/grid of `JourneyCard` components (only shown when nearby mode is OFF)
- **API calls:** `GET /journeys?tag=&q=` on mount + on filter change; `GET /journeys?lat=&lng=&radius_km=5` when nearby mode is active
- **Props:** none (reads from `journeyStore`)
- **Navigation:** Tap card → `JourneyDetailScreen`

#### JourneyDetailScreen
- **Purpose:** Full journey info + start CTA
- **Layout:**
  - Hero image (full-width, 250px height) with gradient overlay + title
  - Meta row: duration badge, tag chips
  - Description paragraph
  - "Stops" section: numbered list of `StopPreviewCard` components (order, title, location label, time)
  - Sticky bottom CTA bar: "Fork" (tertiary/secondary, left) + "Start Solo" (primary) + "Start Group" (secondary) buttons
- **Fork button:** Calls `forkJourney()` API with loading state (disables button during call) → on success, navigates to `JourneyEditorScreen` in draft mode with pre-filled data from the original journey. Shows label "Forked from [original title]". On error, shows toast with error message. User can modify and publish.
- **API calls:** `GET /journeys/:id` on mount; `POST /journeys/:id/fork` on fork
- **Props:** `route.params: { journeyId: string }`
- **Navigation:** "Start Solo" → creates solo session → `ActiveJourneyScreen`  
  "Start Group" → creates group session → shows `InviteFriendsModal` → `ActiveJourneyScreen`  
  "Fork" → `JourneyEditorScreen` (draft mode, forked journey)

### CreateStack

#### CreateHomeScreen
- **Purpose:** Empty state + entry point for journey creation
- **Layout:** Centered illustration + "Create your first journey" headline + CTA button
- **Navigation:** CTA → `JourneyEditorScreen`

#### JourneyEditorScreen
- **Purpose:** Multi-step journey creation/editing
- **Steps (managed by local `useState<number>(0)`):**
  1. **Details** — title, description, cover image picker, tags input, duration label
  2. **Stops** — add/edit/reorder `StopEditorCard` components (drag to reorder)
  3. **Preview** — read-only preview of the full journey as user will see it
  4. **Publish** — confirm + publish toggle
- **Fork mode (MAJOR):** When navigated from `JourneyDetailScreen` via Fork button, `route.params` includes `{ forkedFromId: string }`. The journey data is pre-filled from the original. Header/subtitle shows "Forked from [original title]". On save, creates a new draft journey with `forked_from_id` set.
- **Components:** `ImagePicker` (cover), `TagInput`, `DurationPicker`, `StopEditorCard`, `DraggableStopList`, `PreviewJourneyCard`
- **API calls:** 
  - Create: `POST /journeys`
  - Edit: `PUT /journeys/:id`
- **Props:** `route.params?: { journeyId?: string; forkedFromId?: string }` (edit or fork mode)
- **Navigation:** Save/publish → `MyJourneysTab`

### MyJourneysStack

#### MyJourneysScreen
- **Purpose:** User's created journeys + participation history
- **Layout:** `TopTabNavigator` (React Navigation material top tabs):
  - **"Created" tab:** List of `JourneyCard` (own journeys) — `GET /users/me/journeys`
  - **"History" tab:** List of `CompletedJourneyCard` — `GET /users/me/history`
- **API calls:** `GET /users/:id/journeys` (using `authStore.user.id`), `GET /users/me/history`
- **Navigation:** Tap → `JourneyDetailScreen` (created) or `JourneyHistoryDetailScreen` (history)

#### JourneyHistoryDetailScreen
- **Purpose:** View a completed journey from history
- **Layout:** Similar to `JourneyDetailScreen` but:
  - Read-only, no CTA
  - Shows completion date, total duration
  - Photo gallery of all photos taken during the journey
  - Personal notes (if any)
  - **"View Recap" button** → `JourneyRecapScreen` (MAJOR: enhanced recap with Memory Reel)
- **API calls:** `GET /users/me/history/:completionId` (or include photos in history list)
- **Props:** `route.params: { completionId: string }`

#### JourneyRecapScreen (MAJOR: new screen)
- **Purpose:** Beautiful recap of a completed journey, accessible from history detail
- **Layout:**
  - Hero image (journey cover, full-width), title overlay, completion date, duration
  - Stats row: completion date, total time, stops completed, photos taken
  - **Per-stop sections (chronological):** stop title, location name, map pin icon, horizontal scroll of photos from that stop, mood/atmosphere summary (derived from most-common reaction emoji across members)
  - **Mood/atmosphere from reactions:** aggregate all group members' reactions per stop; display dominant emoji as the "vibe" for that stop section
  - **"Play Memory Reel" button (MAJOR):** tapping opens a fullscreen slideshow of ALL completion photos in chronological order (per stop). Auto-advances every 3 seconds with fade transition. No music. Close button to exit.
  - **"Share" button (MAJOR):** copies public recap URL to clipboard (`GET /recap/:completionId` returns a `public_url`). Shows toast "Link copied!"
  - "Fork This Journey" button — calls `forkJourney()` → navigates to `JourneyEditorScreen` in draft mode
- **Memory Reel implementation:** Fullscreen modal/overlay. `FlatList` of completion photos, `onViewableItemsChanged` with 3s `setInterval` timer to auto-advance. `Animated.View` with `opacity` fade-in/fade-out (300ms) on each transition. No audio.
- **API calls:** `GET /recap/:completionId` (public endpoint — recap data included)
- **Props:** `route.params: { completionId: string }`
- **Navigation:** Back → `JourneyHistoryDetailScreen`

### Distance Validation

**NOTE:** Distance validation is performed **server-side** to prevent GPS spoofing. The client sends raw coordinates to the server, which runs the Haversine formula. The client-side `haversineDistance` utility is kept for UI feedback (showing "Xm away") but is NOT used for security decisions.

```ts
// src/utils/distance.ts
// Haversine formula for distance between two lat/lng points
// Used for UI display only — NOT for security decisions

export function haversineDistance(
  lat1: number,
  lng1: number,
  lat2: number,
  lng2: number
): number {
  const R = 6371000; // Earth radius in meters
  const toRad = (deg: number) => (deg * Math.PI) / 180;

  const dLat = toRad(lat2 - lat1);
  const dLng = toRad(lng2 - lng1);

  const a =
    Math.sin(dLat / 2) * Math.sin(dLat / 2) +
    Math.cos(toRad(lat1)) *
      Math.cos(toRad(lat2)) *
      Math.sin(dLng / 2) *
      Math.sin(dLng / 2);

  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  return R * c;
}

// UI-only constant — server enforces the actual radius
export const CHECK_IN_RADIUS_METERS = 100;
```

### Check-In Hook

```ts
// src/hooks/useGeolocation.ts
import Geolocation from '@react-native-community/geolocation';
import { useState, useCallback } from 'react';
import { PermissionsAndroid, Platform } from 'react-native';

interface UseGeolocationReturn {
  location: { lat: number; lng: number } | null;
  error: string | null;
  isLoading: boolean;
  requestPermission: () => Promise<boolean>;
  getCurrentLocation: () => Promise<{ lat: number; lng: number } | null>;
}

export function useGeolocation() {
  const [location, setLocation] = useState<{ lat: number; lng: number } | null>(null);
  const [error, setError] = useState<string | null>(null);
  const [isLoading, setIsLoading] = useState(false);

  const requestPermission = useCallback(async (): Promise<boolean> => {
    if (Platform.OS === 'android') {
      const granted = await PermissionsAndroid.request(
        PermissionsAndroid.PERMISSIONS.ACCESS_FINE_LOCATION,
        {
          title: 'Location Permission',
          message: 'JourneyTogether needs your location to check in at stops.',
          buttonNeutral: 'Ask Me Later',
          buttonNegative: 'Cancel',
          buttonPositive: 'OK',
        }
      );
      return granted === PermissionsAndroid.RESULTS.GRANTED;
    }
    // iOS: Geolocation.requestAuthorization pops the system dialog automatically
    return true;
  }, []);

  const getCurrentLocation = useCallback(async (): Promise<{ lat: number; lng: number } | null> => {
    setIsLoading(true);
    setError(null);

    return new Promise((resolve) => {
      Geolocation.getCurrentPosition(
        (position) => {
          const { latitude, longitude } = position.coords;
          setLocation({ lat: latitude, lng: longitude });
          setIsLoading(false);
          resolve({ lat: latitude, lng: longitude });
        },
        (err) => {
          setError(err.message);
          setIsLoading(false);
          resolve(null);
        },
        {
          enableHighAccuracy: true,
          timeout: 15000,
          maximumAge: 10000,
        }
      );
    });
  }, []);

  return { location, error, isLoading, requestPermission, getCurrentLocation };
}
```

## 8. WebSocket Client

**⚠️ CRITICAL: Message schema.** The backend uses `{ event, data }` (not `{ type, payload }`). All WebSocket messages between frontend and backend must use the `{ event, data }` format.

### Architecture

One WebSocket connection per active session. Connection lifecycle managed in `sessionStore`.

```ts
// src/api/websocket.ts
import { WS_BASE_URL } from '@/config/env';
import { sessionStore } from '@/stores/sessionStore';
import { authStore } from '@/stores/authStore';

class WebSocketClient {
  private ws: WebSocket | null = null;
  private sessionId: string | null = null;
  private reconnectAttempts = 0;
  private maxReconnectAttempts = 5;
  private reconnectDelay = 1000; // ms

  connect(sessionId: string) {
    this.sessionId = sessionId;
    const token = authStore.getState().token;
    if (!token) return;

    // Backend expects { event, data } messages — NOT { type, payload }
    // Auth via Authorization header (token passed separately in connection setup)
    const url = `${WS_BASE_URL}/ws/session/${sessionId}`;
    this.ws = new WebSocket(url, [], {
      headers: {
        Authorization: `Bearer ${token}`,
      },
    });

    this.ws.onopen = () => {
      console.log('[WS] Connected to session:', sessionId);
      this.reconnectAttempts = 0;
    };

    this.ws.onmessage = (event) => {
      try {
        // Backend sends: { event: string, data: any }
        const message: WSMessage = JSON.parse(event.data);
        sessionStore.handleWSMessage(message);
      } catch (e) {
        console.error('[WS] Failed to parse message:', e);
      }
    };

    this.ws.onerror = (error) => {
      console.error('[WS] Error:', error);
    };

    this.ws.onclose = (event) => {
      console.log('[WS] Closed:', event.code, event.reason);
      this.ws = null;
      // Attempt reconnect if unexpected close
      if (event.code !== 1000 && this.reconnectAttempts < this.maxReconnectAttempts) {
        this.reconnectAttempts++;
        setTimeout(() => this.connect(this.sessionId!), this.reconnectDelay * this.reconnectAttempts);
      }
    };
  }

  disconnect() {
    if (this.ws) {
      this.ws.close(1000, 'User left session');
      this.ws = null;
    }
    this.sessionId = null;
    this.reconnectAttempts = 0;
  }

  // All outgoing messages use { event, data } schema to match backend
  send(message: { event: string; data: any }) {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(message));
    }
  }
}

export const wsClient = new WebSocketClient();
```

### Integration in sessionStore

```ts
// Part of sessionStore.actions — connectWS/disconnectWS

connectWS(sessionId: string) {
  wsClient.connect(sessionId);
}

disconnectWS() {
  wsClient.disconnect();
}

handleWSMessage(message: WSMessage) {
  const session = this.activeSession;
  if (!session) return;

  switch (message.event) {
    case 'session_state':
      // Full session sync from server — update all state
      this.setState((state) => {
        if (!state.activeSession) return state;
        return {
          ...state,
          activeSession: {
            ...state.activeSession,
            members: message.data.members,
            current_stop_index: message.data.session.current_stop_index,
            status: message.data.session.status,
            myProgress: {
              ...state.activeSession.myProgress,
              photos: message.data.photos,
            },
          },
        };
      });
      break;

    case 'stop_completed':
      // Update the member's completed_stop_ids and check if session should advance
      this.setState((state) => {
        if (!state.activeSession) return state;
        const { user_id, stop_id, stop_index, all_at_stop, total_members, journey_completed, new_current_stop_index } = message.data;
        const updatedMembers = state.activeSession.members.map((m) =>
          m.user_id === user_id
            ? { ...m, completed_stop_ids: [...m.completed_stop_ids, stop_id], current_stop_index: stop_index }
            : m
        );
        return {
          ...state,
          activeSession: {
            ...state.activeSession,
            members: updatedMembers,
            current_stop_index: new_current_stop_index,
            status: journey_completed ? 'completed' : state.activeSession.status,
          },
        };
      });
      if (message.data.journey_completed) {
        uiStore.getState().showToast('Journey complete! 🎉', 'success');
      } else {
        uiStore.getState().showToast('A member checked in!', 'info');
      }
      break;

    case 'member_joined':
      this.setState((state) => {
        if (!state.activeSession) return state;
        const exists = state.activeSession.members.some((m) => m.user_id === message.data.user_id);
        if (exists) return state;
        return {
          ...state,
          activeSession: {
            ...state.activeSession,
            members: [...state.activeSession.members, message.data],
          },
        };
      });
      uiStore.getState().showToast(`${message.data.display_name} joined the journey!`, 'info');
      break;

    case 'member_left':
      this.setState((state) => {
        if (!state.activeSession) return state;
        return {
          ...state,
          activeSession: {
            ...state.activeSession,
            members: state.activeSession.members.filter((m) => m.user_id !== message.data.user_id),
          },
        };
      });
      break;

    case 'journey_completed':
      this.setState((state) => {
        if (!state.activeSession) return state;
        return { ...state, activeSession: { ...state.activeSession, status: 'completed' } };
      });
      // Navigate to JourneyRecapScreen with the completionId from the event
      // navigationRef.navigate('JourneyRecap', { completionId: message.data.completionId });
      uiStore.getState().showToast('Journey complete! 🎉', 'success');
      break;

    case 'photo_added':
      this.setState((state) => {
        if (!state.activeSession) return state;
        const currentStopPhotos = state.activeSession.myProgress.photos;
        return {
          ...state,
          activeSession: {
            ...state.activeSession,
            myProgress: {
              ...state.activeSession.myProgress,
              photos: [...currentStopPhotos, message.data.photo],
            },
          },
        };
      });
      break;

    case 'reaction_added':  // MAJOR
      this.setState((state) => {
        if (!state.activeSession) return state;
        return {
          ...state,
          activeSession: {
            ...state.activeSession,
            myProgress: {
              ...state.activeSession.myProgress,
              reactions: {
                ...state.activeSession.myProgress.reactions,
                [message.data.stop_id]: message.data.emoji,
              },
            },
          },
        };
      });
      break;

    case 'session_paused':  // MAJOR
      this.setState((state) => {
        if (!state.activeSession) return state;
        return { ...state, activeSession: { ...state.activeSession, status: 'paused' } };
      });
      uiStore.getState().showToast('Journey paused', 'info');
      break;

    case 'session_resumed':  // MAJOR
      this.setState((state) => {
        if (!state.activeSession) return state;
        return { ...state, activeSession: { ...state.activeSession, status: 'active' } };
      });
      uiStore.getState().showToast('Journey resumed!', 'info');
      break;

    case 'error':
      uiStore.getState().showToast(`Error: ${message.data.message}`, 'error');
      break;
  }
}
```

**Note on reconnection:** If the WS connection drops mid-journey, the app falls back to polling `GET /sessions/:id` every 10 seconds until the WS reconnects.

### Axios Interceptor

```ts
// src/api/client.ts
import axios, { AxiosInstance, InternalAxiosRequestConfig } from 'axios';
import { API_BASE_URL } from '@/config/env';
import { authStore } from '@/stores/authStore';

const client: AxiosInstance = axios.create({
  baseURL: API_BASE_URL,
  timeout: 15000,
  headers: { 'Content-Type': 'application/json' },
});

// Request interceptor: attach JWT
client.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = authStore.getState().token;
    if (token && config.headers) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// Response interceptor: handle 401 → logout
client.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      // Token expired or invalid — force logout
      await authStore.getState().logout();
    }
    return Promise.reject(error);
  }
);

export default client;
```

### Auth Store Persistence

```ts
// src/stores/authStore.ts (persist slice)
import AsyncStorage from '@react-native-async-storage/async-storage';

interface PersistedAuth {
  token: string | null;
  user: User | null;
}

export const authStore = create<AuthState>()(
  persist(
    (set, get) => ({
      user: null,
      token: null,
      isLoading: true,
      isAuthenticated: false,

      login: async (email, password) => {
        const res = await client.post<{ token: string; user: User }>('/auth/login', { email, password });
        const { token, user } = res.data;
        set({ token, user, isAuthenticated: true });
      },

      register: async (email, password, displayName) => {
        const res = await client.post<{ token: string; user: User }>('/auth/register', {
          email, password, display_name: displayName,
        });
        const { token, user } = res.data;
        set({ token, user, isAuthenticated: true });
      },

      logout: async () => {
        set({ token: null, user: null, isAuthenticated: false });
        await AsyncStorage.removeItem(AUTH_TOKEN_KEY);
        await AsyncStorage.removeItem(AUTH_USER_KEY);
        sessionStore.getState().disconnectWS();
        sessionStore.getState().leaveSession();
      },

      loadToken: async () => {
        set({ isLoading: true });
        const token = await AsyncStorage.getItem(AUTH_TOKEN_KEY);
        const userJson = await AsyncStorage.getItem(AUTH_USER_KEY);
        if (token && userJson) {
          const user = JSON.parse(userJson) as User;
          set({ token, user, isAuthenticated: true });
        }
        set({ isLoading: false });
      },

      updateProfile: async (updates) => {
        const res = await client.put<User>('/users/me', updates);
        set({ user: res.data });
      },

      uploadAvatar: async (uri: string) => {
        // MAJOR: upload avatar using uploadAvatar utility
        const avatarUrl = await uploadAvatar(uri);
        if (avatarUrl) {
          await get().updateProfile({ avatar_url: avatarUrl });
        }
      },

      setUser: (user) => set({ user }),
    }),
    {
      name: 'jt-auth',
      storage: createJSONStorage(() => AsyncStorage),
      partialize: (state) => ({ token: state.token, user: state.user, isAuthenticated: state.isAuthenticated }),
    }
  )
);
```

```ts

// 1. Clear token + user from state + AsyncStorage
```

### Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Map library | `react-native-maps` | Native performance, supports markers, polylines, user location |
| Map fallback | WebView + OpenStreetMap | If native maps have build issues on Unraid CI |
| State persistence | Zustand `persist` middleware + AsyncStorage | Token/user persisted; session/UI transient |
| Image upload | Raw `fetch` with `FormData` + `compressImage()` | Avoids axios multipart complexity; compression ensures reasonable file sizes |
| WS reconnection | Exponential backoff (max 5 attempts) | Graceful degradation to polling |
| Navigation | React Navigation 7 | Stable, typed, supports deep linking |
| Forms | `react-hook-form` + `zod` | Type-safe validation, great DX |
| Icons | `react-native-vector-icons` (Ionicons set) | Matches spec's clean, modern aesthetic |
| GPS validation | Server-side Haversine | Prevents GPS spoofing; client sends raw coords, server validates |
